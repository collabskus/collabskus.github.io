Purpose & context
Kushal is developing a Blazor WebAssembly application that displays Kathmandu calendar and time data with sophisticated moon phase calculations. The application is deployed at https://collabskus.github.io/ and serves users who need accurate lunar information including Nepal Sambat tithi data. The project requires extremely high precision in astronomical calculations with real-time updates, combining traditional Nepali calendar systems with modern web technologies. Success is measured by accurate, synchronized display of moon phase data that updates frequently without performance degradation.
Current state
The application has undergone significant debugging and enhancement cycles to resolve critical synchronization and accuracy issues. Recent work focused on fixing timezone handling bugs that caused moon phase discrepancies between mobile and desktop devices, where calculations were off by approximately 5 hours 45 minutes due to improper DateTime handling in the Kathmandu time conversion. The core issue was resolved by ensuring moon phase calculations use UTC time directly rather than timezone-adjusted values. The UI components have been updated to properly display contextual moon phase timing information and Nepal Sambat tithi data that were previously missing from the interface.
Key learnings & principles
Astronomical calculations must always use UTC time as the reference point since moon phases are identical globally at any given UTC instant - timezone adjustments should only be applied for display purposes, never for the underlying calculations. Device-specific network latency can cause subtle variations in time offset calculations, leading to inconsistent results across platforms. The importance of separating data calculation frequency from UI update frequency has been established - moon phase data can be calculated less frequently while still maintaining responsive UI updates. Semantic accuracy in traditional calendar systems requires careful handling of special cases like Purnima/Amavasya rather than oversimplified array mappings.
Approach & patterns
Development follows a systematic debugging approach involving comprehensive code reviews, line-by-line analysis, and verification against external astronomical data sources. Issues are prioritized by user impact, with synchronization problems and accuracy discrepancies taking precedence. The codebase uses a service-oriented architecture separating concerns between time management, astronomical calculations, and UI rendering. Updates are implemented with careful attention to avoiding regressions, often involving targeted fixes to specific methods rather than wholesale rewrites. Performance optimization balances calculation accuracy with update frequency requirements.
Tools & resources
The application is built using Blazor WebAssembly and deployed via GitHub Pages. Astronomical calculations implement the Jean Meeus algorithm from "Astronomical Algorithms" with 60 principal perturbation terms for high precision. The moon phase service uses proper Julian Day Number calculations and ecliptic longitude differences between Moon and Sun positions. External web sources are used for verification of calculated moon phase data to ensure accuracy against established astronomical references.
